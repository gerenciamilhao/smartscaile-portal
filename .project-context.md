# smartscaile-portal

Portal de propostas/diagnósticos para clientes da smartscaile.

## Stack

- Next.js 16 (App Router) + TypeScript
- Tailwind CSS v4
- GSAP (ScrollTrigger + ScrollSmoother) — scroll suave, snap, animações scroll-driven
- Framer Motion — AnimatePresence, micro-interações (hover, pulse, transição locked→unlocked)
- jose (JWT)
- Lucide React (icons)
- Deploy: Vercel (hub.smartscaile.com)

## Build

```bash
npm run build
```

## Dev

```bash
npm run dev
```

## Deploy

Vercel auto-deploy via git push. Domínio: `hub.smartscaile.com`

## Arquitetura

### Experiência Cinematográfica (rota `/`)

Fluxo: PainHero (vermelho, locked) → SlideToUnlock → TokenModal → Hero (teal, unlocked)

- `CinematicExperience.tsx` — orquestrador (state machine: locked → unlocking → unlocked)
- `PainHero.tsx` — cópia fiel da LP, tema vermelho, event stream com falhas
- `Hero.tsx` — cópia fiel da LP, tema teal, server-side tracking ativo
- `SlideToUnlock.tsx` — slider drag, transição vermelho→teal
- `TokenModal.tsx` — modal glassmorphism para inserir token de acesso

### Scroll System (GSAP)

- `GSAPProvider.tsx` — registra ScrollTrigger + ScrollSmoother, wrapper/content structure
- `ScrollSlide.tsx` — slide cinematográfico com crossfade via scrollYProgress (Framer Motion useTransform)
- `ProposalScroll.tsx` — 5 slides de proposta com stagger animations, glassmorphism cards, ambient effects
- `ProgressIndicator.tsx` — dots + rail sincronizados via ScrollTrigger
- `CinematicExperience.tsx` — orquestrador + persistent ambient glow layer
- **Lenis removido** (2026-03-19) — substituído por GSAP ScrollSmoother
- **Framer Motion mantido** apenas para AnimatePresence e micro-interações

### Slide Transition System (CRÍTICO)

Container tall (`1050vh`) com viewport sticky (`100vh`). Slides são `position: absolute` sobrepostos, controlados via `scrollYProgress` (Framer Motion `useTransform`).

**Regras de transição (aprendidas em 2026-03-19):**
- Slides DEVEM ser **transparentes** (sem `background`). O `bg-hero-mesh` no viewport sticky é o background único
- Crossfade overlap: **30% entrada + 30% saída** do range de cada slide (valores menores criam corte visível)
- Drift sutil: **30px entrada, -20px saída** (valores maiores distraem)
- **Ambient glow persistente** no viewport sticky (z-index 0) — elemento fixo teal que cria continuidade visual entre slides
- NUNCA colocar glows individuais por slide (causa artefatos durante crossfade)
- `isLast` flag no último slide: sem fade-out, permanece visível

**Ranges atuais (7 slides em 1050vh):**
```
header:        [0.06, 0.19]
results:       [0.19, 0.33]
goal1 (Escala):[0.33, 0.46]
goal2 (CPA):   [0.46, 0.59]
goal3 (Dados): [0.59, 0.72]
opportunities: [0.72, 0.86]
proposal:      [0.86, 1.00]  (isLast)
```

**Arquitetura de animação (Slide 1 + 2):**
- **Scroll controla entrada** (opacity + y drift via `useTransform`)
- **Loop controla dados** (animações infinitas independentes do scroll)
- `LOOP_S = 6`, `LOOP_TIMES = [0, 0.5, 0.67, 1]` → 3s fill, 1s hold, 2s smooth reset

**Slide 1 — Terminal Typing:**
- Efeito char-by-char com `useState` + `useEffect` + `setTimeout` (25-45ms variação natural)
- Perspectiva 3D: `rotateY(8deg) rotateX(1deg)`, `transformOrigin: 'left center'`
- Tamanho fixo: todas as linhas renderizadas com `visibility: hidden` (evita layout shift)
- `useMemo` nas linhas do terminal para estabilidade de referência
- Glassmorphism, macOS window chrome, line numbers, bottom bar com "live" dot

**Slide 2 — Stape Diagnostic:**
- Ring SVG (180px, stroke 8) com loop animation (`strokeDashoffset`)
- Sub-scores minimalistas: layout horizontal (icon + label + score inline), barra 2px, separadores sutis
- Container sub-scores: `maxWidth: 280px`
- `scoreColor(v)`: `#EF4444` (≤30), `#F59E0B` (31-50), `#77BDAC` (>50)
- Hooks rule: `SubScoreCard` extraído como componente próprio (não pode usar `useTransform` dentro de `.map()`)

**Slide 3 — Goal Escala (counter + scale bar):**
- **Single animation source**: `Date.now() % (LOOP_S * 1000)` via `setInterval(50ms)` — counter e barra derivam do mesmo `loopProgress`
- **4 fases**: up (easeOutCubic, 0→0.42) → hold topo (0.42→0.55) → down (easeInOutCubic, 0.55→0.88) → hold fundo (0.88→1)
- Counter: R$3.000 → R$100.000, fonte mono, `tabular-nums`
- Barra: `max-w-[380px]`, 3px, gradiente teal, glow sutil, labels "atual" / "potencial"
- **Context pills**: 4 pills com animação hold-release (6 keyframes, ~5-8s ciclo, stagger 1.2s, pausa ~1.2s no pico)

**Slide 4 — Goal CPA (tracking ring + CPA counter):**
- **Tracking ring** (120px, stroke 6): versão menor do ring do slide 2, sincronizado com `normalizedValue` (31→100)
- **Cores tracking**: `#EF4444` (≤30) → `#F59E0B` (31-50) → `#60A5FA` azul (51-89) → `#77BDAC` verde (90+)
- **CPA counter**: R$191→R$100, inversamente correlacionado (mesmo `normalizedValue`)
- **Meta badge animado**: border pulse (4s) + glow + dot pulsante
- **Context pills amber**: tom `#D97706`/`#F59E0B` (dados negativos/alerta), mesma animação hold-release
- **`useLoopProgress()` hook**: `requestAnimationFrame` com throttle ~30fps, calcula `normalizedValue` com 4 fases (easeOut up → hold → easeInOut down → hold bottom). Compartilhado entre slides 3 e 4
- **`CPATrackingRing`**: componente isolado com ring SVG, orbs flutuantes, glow breathing

## Regras

- Dados de clientes em `src/data/clients/` (JSON estático)
- Token map em `src/data/token-map.json`
- Auth via JWT (jose) + cookie httpOnly
- `.env` com `JWT_SECRET` obrigatório
- Visual reutiliza design system da LP (smartscaile-lp)
- Dark mode obrigatório
- Componentes cinematográficos são cópias fiéis da LP. Manter paridade visual

## API

- `POST /api/auth` — valida token, seta cookie JWT, retorna `{ success, name, clientId }`
- `GET /api/client-data` — lê cookie JWT, retorna dados do cliente

## Routes

- `/` — experiência cinematográfica (PainHero → Hero)
- `/portal/proposal` — redirect para `/`
- `/portal` — redirect para `/`
