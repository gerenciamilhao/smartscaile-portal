# smartscaile-portal

Portal de propostas/diagnósticos para clientes da smartscaile.

## Stack

- Next.js 16 (App Router) + TypeScript
- Tailwind CSS v4
- Framer Motion
- jose (JWT)
- Lucide React (icons)
- Deploy: Vercel (hub.smartscaile.com)

## Build

```bash
npm run build
```

## Dev

```bash
npm run dev
```

## Deploy

Vercel auto-deploy via git push. Domínio: `hub.smartscaile.com`

## Arquitetura

### Experiência Cinematográfica (rota `/`)

Fluxo: PainHero (vermelho, locked) → SlideToUnlock → TokenModal → Hero (teal, unlocked)

- `CinematicExperience.tsx` — orquestrador (state machine: locked → unlocking → unlocked)
- `PainHero.tsx` — cópia fiel da LP, tema vermelho, event stream com falhas
- `Hero.tsx` — cópia fiel da LP, tema teal, server-side tracking ativo
- `SlideToUnlock.tsx` — slider drag, transição vermelho→teal
- `TokenModal.tsx` — modal glassmorphism para inserir token de acesso

### Design System

Partilhado com a LP (`smartscaile-lp`). Style guide em `website-guidelines/STYLE_GUIDE.md`.

### Componentes de Proposta (existentes, uso futuro)

- `ProposalHeader`, `ProjectedResults`, `DesiresSection`, `Opportunities`, `Proposal`, `ProposalFooter`
- Componentes prontos em `src/components/portal/`

## Regras

- Dados de clientes em `src/data/clients/` (JSON estático)
- Token map em `src/data/token-map.json`
- Auth via JWT (jose) + cookie httpOnly
- `.env` com `JWT_SECRET` obrigatório
- Visual reutiliza design system da LP (smartscaile-lp)
- Dark mode obrigatório
- Componentes cinematográficos são cópias fiéis da LP. Manter paridade visual

## API

- `POST /api/auth` — valida token, seta cookie JWT, retorna `{ success, name, clientId }`
- `GET /api/client-data` — lê cookie JWT, retorna dados do cliente

## Routes

- `/` — experiência cinematográfica (PainHero → Hero)
- `/portal/proposal` — redirect para `/`
- `/portal` — redirect para `/`
