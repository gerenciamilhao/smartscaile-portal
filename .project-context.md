# smartscaile-portal

Portal de propostas/diagnósticos para clientes da smartscaile.

## Stack

- Next.js 16 (App Router) + TypeScript
- Tailwind CSS v4
- GSAP (ScrollTrigger + ScrollSmoother) — scroll suave, snap, animações scroll-driven
- Framer Motion — AnimatePresence, micro-interações (hover, pulse, transição locked→unlocked)
- jose (JWT)
- Lucide React (icons)
- Deploy: Vercel (hub.smartscaile.com)

## Build

```bash
npm run build
```

## Dev

```bash
npm run dev
```

## Deploy

Vercel auto-deploy via git push. Domínio: `hub.smartscaile.com`

## Arquitetura

### Experiência Cinematográfica (rota `/`)

Fluxo: PainHero (vermelho, locked) → SlideToUnlock → TokenModal → Hero (teal, unlocked)

- `CinematicExperience.tsx` — orquestrador (state machine: locked → unlocking → unlocked)
- `PainHero.tsx` — cópia fiel da LP, tema vermelho, event stream com falhas
- `Hero.tsx` — cópia fiel da LP, tema teal, server-side tracking ativo
- `SlideToUnlock.tsx` — slider drag, transição vermelho→teal
- `TokenModal.tsx` — modal glassmorphism para inserir token de acesso

### Scroll System (GSAP)

- `GSAPProvider.tsx` — registra ScrollTrigger + ScrollSmoother, wrapper/content structure
- `ScrollSlide.tsx` — slide cinematográfico com crossfade via scrollYProgress (Framer Motion useTransform)
- `ProposalScroll.tsx` — 5 slides de proposta com stagger animations, glassmorphism cards, ambient effects
- `ProgressIndicator.tsx` — dots + rail sincronizados via ScrollTrigger
- `CinematicExperience.tsx` — orquestrador + persistent ambient glow layer
- **Lenis removido** (2026-03-19) — substituído por GSAP ScrollSmoother
- **Framer Motion mantido** apenas para AnimatePresence e micro-interações

### Slide Transition System (CRÍTICO)

Container tall (`600vh`) com viewport sticky (`100vh`). Slides são `position: absolute` sobrepostos, controlados via `scrollYProgress` (Framer Motion `useTransform`).

**Regras de transição (aprendidas em 2026-03-19):**
- Slides DEVEM ser **transparentes** (sem `background`). O `bg-hero-mesh` no viewport sticky é o background único
- Crossfade overlap: **30% entrada + 30% saída** do range de cada slide (valores menores criam corte visível)
- Drift sutil: **30px entrada, -20px saída** (valores maiores distraem)
- **Ambient glow persistente** no viewport sticky (z-index 0) — elemento fixo teal que cria continuidade visual entre slides
- NUNCA colocar glows individuais por slide (causa artefatos durante crossfade)
- `isLast` flag no último slide: sem fade-out, permanece visível

**Ranges atuais (5 slides em 600vh):**
```
header:        [0.10, 0.28]
results:       [0.28, 0.46]
desires:       [0.46, 0.64]
opportunities: [0.64, 0.82]
proposal:      [0.82, 1.00]  (isLast)
```

## Regras

- Dados de clientes em `src/data/clients/` (JSON estático)
- Token map em `src/data/token-map.json`
- Auth via JWT (jose) + cookie httpOnly
- `.env` com `JWT_SECRET` obrigatório
- Visual reutiliza design system da LP (smartscaile-lp)
- Dark mode obrigatório
- Componentes cinematográficos são cópias fiéis da LP. Manter paridade visual

## API

- `POST /api/auth` — valida token, seta cookie JWT, retorna `{ success, name, clientId }`
- `GET /api/client-data` — lê cookie JWT, retorna dados do cliente

## Routes

- `/` — experiência cinematográfica (PainHero → Hero)
- `/portal/proposal` — redirect para `/`
- `/portal` — redirect para `/`
