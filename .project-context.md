# smartscaile-portal

Portal de propostas/diagnósticos para clientes da smartscaile.

## Stack

- Next.js 16 (App Router) + TypeScript
- Tailwind CSS v4
- Lenis — smooth scroll com inércia
- GSAP (ScrollTrigger) — integrado com Lenis para eventos de scroll
- Framer Motion — AnimatePresence, micro-interações (hover, pulse, transição locked→unlocked)
- jose (JWT)
- Lucide React (icons)
- Deploy: Vercel (propostas.smartscaile.com)

## Build

```bash
npm run build
```

## Dev

```bash
npm run dev
```

## Deploy

Vercel auto-deploy via git push. Domínio: `propostas.smartscaile.com`

## Arquitetura

### Experiência Cinematográfica (rota `/{cliente}`)

Fluxo: PainHero (vermelho, locked) → HoldToUnlock → TokenModal → HeaderSlide (proposta direto)

- `CinematicExperience.tsx` — orquestrador (state machine: locked → unlocked). AnimatePresence `mode="wait"` com transições 0.3s
- `PainHero.tsx` — tema vermelho, event stream com falhas, HoldToUnlock. Zoom via `visualViewport.height` (mobile Safari)
- `Hero.tsx` — tema teal (NÃO importado, mantido para uso futuro)
- `TokenModal.tsx` — modal limpo: fundo preto 98% opacidade, card transparente, logo smartscaile 15% opacidade, input 16px (anti-zoom Safari)
- `ScrollSlide.tsx` — crossfade com `isFirst` (sem fade-in) e `isLast` (sem fade-out). Desktop scale capped a 1.20

### Scroll System

- `ScrollSlide.tsx` — slide cinematográfico com crossfade via scrollYProgress (Framer Motion useTransform). `scaleFactor = Math.min(innerHeight/800, 1.20)` — mobile ~1.0, desktop capped 1.20
- `ProposalScroll.tsx` — 8 slides de proposta, sem ambient effects (removidos para performance)
- `ProgressIndicator.tsx` — dots + rail sincronizados via scrollYProgress
- **Framer Motion** — AnimatePresence, micro-interações, scroll-driven animations (useScroll/useTransform)

### Slide Transition System (CRÍTICO)

Container tall (`1200vh`) com viewport sticky (`100vh`). Slides são `position: absolute` sobrepostos, controlados via `scrollYProgress` (Framer Motion `useTransform`).

**Regras de transição (aprendidas em 2026-03-19):**
- Slides DEVEM ser **transparentes** (sem `background`). O `bg-hero-mesh` no viewport sticky é o background único
- Crossfade overlap: **30% entrada + 30% saída** do range de cada slide (valores menores criam corte visível)
- Drift sutil: **30px entrada, -20px saída** (valores maiores distraem)
- Efeitos pesados (blur, backdropFilter, animated glows, floating orbs) foram removidos para performance
- NUNCA colocar glows individuais por slide (causa artefatos durante crossfade)
- `isFirst` flag no primeiro slide: sem fade-in, conteúdo estático no load. LazyContent checa valor inicial no mount
- `isLast` flag no último slide: sem fade-out, permanece visível

**Ranges atuais (8 slides em 1200vh, header começa do 0 — sem Hero):**
```
header:        [0.000, 0.125]  (isFirst — sem fade-in)
results:       [0.125, 0.250]
goal1 (Escala):[0.250, 0.375]
goal2 (CPA):   [0.375, 0.500]
goal3 (Dados): [0.500, 0.625]
opportunities: [0.625, 0.750]
pricing:       [0.750, 0.875]
proposal:      [0.875, 1.000]  (isLast)
```

**Arquitetura de animação (Slide 1 + 2):**
- **Scroll controla entrada** (opacity + y drift via `useTransform`)
- **Loop controla dados** (animações infinitas independentes do scroll)
- `LOOP_S = 6`, `LOOP_TIMES = [0, 0.5, 0.67, 1]` → 3s fill, 1s hold, 2s smooth reset

**Slide 1 — Terminal Typing (isFirst — conteúdo estático no load):**
- Sem animações de entrada scroll-driven. Todos os elementos visíveis imediatamente
- Efeito char-by-char com `useState` + `useEffect` + `setTimeout` (25-45ms variação natural)
- Perspectiva 3D: `rotateY(8deg) rotateX(1deg)`, `transformOrigin: 'left center'`
- Tamanho fixo: todas as linhas renderizadas com `visibility: hidden` (evita layout shift)
- Mouse scroll indicator animado no footer
- macOS window chrome, line numbers, bottom bar com "live" dot

**Slide 2 — Stape Diagnostic:**
- Ring SVG (180px, stroke 8) com loop animation (`strokeDashoffset`)
- Sub-scores minimalistas: layout horizontal (icon + label + score inline), barra 2px, separadores sutis
- Container sub-scores: `maxWidth: 280px`
- `scoreColor(v)`: `#EF4444` (≤30), `#F59E0B` (31-50), `#77BDAC` (>50)
- Hooks rule: `SubScoreCard` extraído como componente próprio (não pode usar `useTransform` dentro de `.map()`)

**Slide 3 — Goal Escala (counter + scale bar):**
- **Single animation source**: `Date.now() % (LOOP_S * 1000)` via `setInterval(50ms)` — counter e barra derivam do mesmo `loopProgress`
- **4 fases**: up (easeOutCubic, 0→0.42) → hold topo (0.42→0.55) → down (easeInOutCubic, 0.55→0.88) → hold fundo (0.88→1)
- Counter: R$3.000 → R$100.000, fonte mono, `tabular-nums`
- Barra: `max-w-[380px]`, 3px, gradiente teal, glow sutil, labels "atual" / "potencial"
- **Context pills**: 4 pills com animação hold-release (6 keyframes, ~5-8s ciclo, stagger 1.2s, pausa ~1.2s no pico)

**Slide 4 — Goal CPA (tracking ring + CPA counter):**
- **Tracking ring** (120px, stroke 6): versão menor do ring do slide 2, sincronizado com `normalizedValue` (31→100)
- **Cores tracking**: `#EF4444` (≤30) → `#F59E0B` (31-50) → `#60A5FA` azul (51-89) → `#77BDAC` verde (90+)
- **CPA counter**: R$191→R$100, inversamente correlacionado (mesmo `normalizedValue`)
- **Meta badge animado**: border pulse (4s) + glow + dot pulsante
- **Context pills amber**: tom `#D97706`/`#F59E0B` (dados negativos/alerta), mesma animação hold-release
- **`useLoopProgress()` hook**: `requestAnimationFrame` com throttle ~30fps, calcula `normalizedValue` com 4 fases (easeOut up → hold → easeInOut down → hold bottom). Compartilhado entre slides 3 e 4
- **`CPATrackingRing`**: componente isolado com ring SVG (orbs e glow removidos)

**Slide 5 — Goal Dados (loss calculator + split bar):**
- **Loss calculator**: ledger minimalista (max-w-[300px]) com 3 rows — investimento fixo, perda diária animada, perda mensal (hero)
- **`lerpRGB3()` helper**: interpolação RGB contínua entre 3 stops — teal `(119,189,172)` → amber `(245,158,11)` → red `(239,68,68)`
- **Cor smooth**: todos os elementos da calculadora mudam cor frame-a-frame via `normalizedValue` (sem steps discretos, sem CSS transition)
- **Split bar** (2px): porção teal (visível, encolhe 100→70%) + porção loss (cresce 0→30%), cor interpolada
- **Labels**: "visível" / "perdido" (0.4rem, opacity 0.4)
- **Hierarquia**: investimento = contexto (`#4B5563`/`#6B7280`, fw400), perda diária = secondary (`lossRgba(0.5)`), perda mensal = hero (`clamp(1rem, 3.5vw, 1.25rem)`, fw600)
- **Dot**: 3px, opacity proporcional ao `normalizedValue`
- **Red pills**: "Sem CAPI", "Cookie 1 dia", "100% client-side" — mesma animação hold-release, tom `#EF4444`
- **`useLoopProgress()` compartilhado** entre slides 3, 4 e 5

## Regras

- Dados de clientes em `src/data/clients/` (JSON estático)
- Token map em `src/data/token-map.json` (tokens case-sensitive)
- Auth sem sessão — token validado a cada visita, sem cookies/JWT
- `.env` com `JWT_SECRET` não é mais necessário
- Visual reutiliza design system da LP (smartscaile-lp)
- Dark mode obrigatório
- Componentes cinematográficos são cópias fiéis da LP. Manter paridade visual
- WhatsApp CTA: +351934157309

### Regras de Overflow (CRÍTICO)
- **NUNCA** colocar `overflow-x: hidden` no `<html>` ou `<body>` — quebra `position: sticky` e `scrollYProgress` do Framer Motion
- Conter overflow no nível do **componente** (wrapper do card, section)
- PainHero já tem `overflow: hidden` na section + wrapper do card 3D

### Regras de Mobile Safari
- Inputs DEVEM ter `fontSize: 16px` mínimo — Safari faz auto-zoom em inputs menores
- Modais: fundo preto opaco (`rgba(0,0,0,0.98)`), card transparente, sem gradientes/borders decorativos
- Zoom do PainHero: `visualViewport.height` (não `window.innerHeight`) para descontar a barra de endereço
- NÃO usar listener `resize` no visualViewport para zoom (causa efeito estranho no pull-to-refresh)

### Regras de Transição
- `AnimatePresence mode="wait"` (NÃO `sync` — sync causa layout de 1200vh durante crossfade)
- Transições rápidas: 0.3s fade out / 0.3s fade in
- PainHero exit: só `opacity: 0` (sem scale, sem blur)
- Botão sair: scroll para topo primeiro, depois troca estado com 100ms delay
- Sempre começa locked — toda visita mostra PainHero → token obrigatório

## API

- `POST /api/auth` — valida token (case-sensitive), retorna `{ success, name, clientId }`. Sem cookie.
- `GET /api/client-data?cliente={slug}` — retorna dados do cliente pelo slug
